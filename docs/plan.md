# 相控阵 FMC 加速采集：学习式稀疏重建的初步验证方案

> **适用范围**：仿真数据 · 线阵（Linear Array）· 目标为加速采集
> **方案性质**：可行性验证（初步），用于决定是否值得投入更复杂的主动采集方案
> **版本**：v0.1（初稿）

---

## 0. 文档目的

把"用最新视频/世界模型思路加速 FMC 采集"这一想法，收敛成一套**可执行、可证伪**的初步验证流程。核心问题只有一句：

> **只发射 K 个阵元（K < N），能不能重建出与全采集一样能用的 TFM 图像？K 压到多小开始崩？**

本方案给出数据生成、任务定义、模型、必打基线、评价指标、分阶段里程碑与风险清单，目标是先证明（或证伪）"学习式稀疏重建确实能加速"，再谈更进一步的主动感知。

---

## 1. 背景与核心洞察

### 1.1 FMC / TFM 简述

全矩阵采集（FMC）让线阵中每个阵元依次发射、全阵列接收，得到 `N×N` 条时域 A-scan，构成数据立方体 `A[tx, rx, t]`。总聚焦法（TFM）对成像区域每个网格点做物理驱动的"延迟求和"波束合成，得到高分辨缺陷图像。FMC→TFM 是一个**已知且可微的线性算子**——这一点后面会反复用到。

### 1.2 加速的正确旋钮：发射次数，而非 A-scan 总数

这是整个方案成立的前提：

- FMC 的采集时间**正比于发射次数**，而非 A-scan 总数 `N²`。
- 每发射一枪，全阵列并行接收（几乎免费）；真正受限的是"打完一枪要等介质安静下来才能打下一枪"。
- 因此真正该动的旋钮是 **sparse-Tx**：只发射 `K` 个阵元、每枪仍全阵列接收，得到 `K×N` 条 A-scan；再把未发射的"行"补全成完整 `N×N×T` 立方体，最后照常做 TFM。
- **加速比 ≈ N / K**，且直接对应墙钟时间。

### 1.3 诚实性约束：加速比相对 HMC 报

线阵满足互易性 `H_ij(f) = H_ji(f)`。工程上半矩阵采集（HMC）已经靠它白拿约 2× 加速。因此本方案声称的加速比一律**相对 HMC** 报告，避免把已有的免费收益算成方法贡献。

### 1.4 可证伪的问题陈述

> 给定 sparse-Tx 采集（K 枪），重建完整 FMC 立方体后做 TFM，
> 在容差范围内（缺陷检出率、定量误差、伪像水平）能达到的**最大加速比**是多少？
> 该加速比是否显著优于"朴素子采样 + TFM"与"低秩矩阵补全"等廉价基线？

## 2. 数据与仿真设置

### 2.1 前向求解器与几何

- 求解器任选其一：k-Wave / Pogo / CIVA / FEM / 2D FDTD。
- 几何先固定为**平面试块**，把复杂度集中在缺陷上（曲面、楔块等留待后续）。
- 阵列规模以 `N = 64` 作为跑例（按实际阵列调整），时域长度 `T ≈ 1000–3000` 采样点。

### 2.2 样本分布

| 维度 | 主集合（先做） | 难集合 / OOD（压力测试） |
|---|---|---|
| 缺陷类型 | 侧钻孔 SDH | 裂纹类（带取向） |
| 缺陷参数 | 变直径、变深度、变横向位置 | 训练未见的尺寸/位置组合 |
| 介质 | 声速、晶粒噪声小范围扰动 | 换一档声速做 OOD |

### 2.3 真值与数据划分（铁律）

- 每个试块生成**完整 FMC** 作为 ground truth（仿真的优势：真值可得）。
- **按缺陷构型划分训练/验证/测试，绝不按随机 (tx,rx) 对划分**——否则等于数据泄漏。
- 测试集必须包含训练里**没见过的**缺陷位置/尺寸/类型；另设一个换声速的 OOD 集，用来看泛化而非插值。
- 规模：初版几百到几千个试块即可证可行性，先求闭环跑通，再谈扩量。

---

## 3. 任务定义与初版模型

### 3.1 任务：有监督 sparse-Tx 重建

- **输入**：`K` 个发射行，形状 `K×N×T`。
- **输出**：完整立方体 `N×N×T`。
- **理由**：仿真有完整真值，有监督是最干净的可行性信号。先证明"学习式补全到底行不行"，再谈自监督（见阶段 2）。
- 选择"重建立方体"而非"直接出 TFM 图"，是为了与具体成像配方**解耦**，更便于诊断，也能复用现有 TFM 工具链。

### 3.2 两个立刻提升可行性的工程选择

1. **到频域去做。** 只取信号带宽内的若干频点，每个频点是一个 `N×N` 复矩阵 `M_f`。维度直接砍掉一两个数量级；而且因物理自由度有限，`M_f` 近似**低秩**，这对学习和经典基线都有利。
2. **互易性当硬约束。** 强制 `M_f = M_fᵀ`（对称化）。这是免费、确定性的先验，等价于白送一半信息。

### 3.3 模型与损失（初版别过度设计）

- **网络**：在 `(tx, rx, freq)` 上的 3D U-Net，或带通道注意力的等价结构即可，第一版不追求花哨。
- **损失（必须相位敏感）**：频域复数 L2，或时域 RF 信号 L2。
  **绝对不要只用包络损失**——相位一丢，后续相干波束合成全废。
- **可选物理项**：可微 TFM 一致性损失——把重建立方体过一遍延迟求和，与真值 TFM 对齐。

## 4. 对照基线（决定可信度的一节）

学习方法只有打赢下列廉价且强的基线，才算真有价值。"AI+物理"类工作常败于基线太弱。

1. **朴素子采样 + TFM**：只发 `K` 枪，直接用这 `K` 行做 TFM，不做任何重建。
   作用：量化 TFM 随发射数减少的退化下界（栅瓣、信噪比下降）。
2. **线性插值 / 压缩感知（CS）**：跨 tx 插值，或在稀疏基下做 L1 恢复（经典"压缩 FMC"路线）。
3. **低秩矩阵补全**：逐频点对 `M_f` 做补全。**又便宜又强，是最该忌惮的对手。**
4. **物理基线（可选）**：PINN 式波场外推。

**判据**：若低秩补全已与学习网络打平，则该网络不值得做——这是重要且诚实的检查点，负结果同样有价值。

## 5. 评价指标与核心交付物

重建误差只是代理量，真正要回答的是"**检测能力有没有掉**"。指标分三层：

| 层级 | 指标 |
|---|---|
| 采集成本 | 发射枪数 `K`、加速比 `N/K`（相对 HMC） |
| 重建保真度 | 立方体 NRMSE（**相位敏感**） |
| 下游检测（最关键） | TFM 的 API（阵列性能指标）、缺陷 SNR/对比度、−6 dB 定量误差、栅瓣/伪像水平；检出率 vs 缺陷尺寸；误报率 |

### 核心交付物：质量 vs 加速比曲线

每个方法画一条：横轴加速比、纵轴某检测/定量指标。重点看两件事：

- **拐点**：你的方法在容差内能撑到几倍加速（如定量误差 < 阈值、检出率 > 阈值）。
- **优势**：该拐点比朴素子采样、低秩补全好多少。

## 6. 分阶段里程碑与成功判据

| 阶段 | 内容 | 成功判据 |
|---|---|---|
| **0** | 搭好数据生成、基线、指标流水线；先画出"朴素子采样+TFM"与"低秩补全"两条曲线 | 两条基线曲线产出——它们本身就是研究地图 |
| **1** | 有监督 sparse-Tx 重建，固定均匀采样 | 某加速比下 TFM 缺陷 SNR 与定量在容差内，且明显优于朴素子采样、不输于低秩补全 |
| **2** | 泛化压力测试（OOD 缺陷/声速）+ 物理约束（互易、TFM 一致性）+ 自监督版本 | 在 OOD 集上仍守住容差；自监督版本逼近有监督上界 |
| **3** | 把"发射哪 K 枪"从固定均匀改为**学习/主动选择** | 相同发射预算下，主动采样较固定采样进一步提升加速比 |

> 阶段 2 的自监督是为将来迁到**没有完整真值的实测数据**做准备——这是仿真验证迟早要还的债。
> 阶段 3 才是"发射即动作 / 世界模型"那条新意线的回报，务必等阶段 1 跑通后再上。

## 7. 风险与陷阱清单

- **相位损失会骗人**：只看包络的指标好看，重建却无法相干合成。→ 全程相位敏感。
- **数据泄漏**：按缺陷构型划分，而非随机 (tx,rx) 对。
- **加速比虚高**：一律相对 HMC 报，别把互易性的 2× 算成方法贡献。
- **基线太弱**：必须打低秩补全与 CS。
- **耦合成像配方**：重建立方体而非直接出 TFM 图，便于诊断与复用。
- **Sim-to-real 差距**：当前全为仿真，迁到实测是另一场仗；阶段 2 的自监督 + 物理一致性即为过此坎做准备。
- **过度设计**：初版模型从简，先证可行性，再迭代结构。

---

## 8. 建议的下一步

- **若想先动手验证可行性**：从阶段 1 的有监督 sparse-Tx 重建做起（固定均匀采样、频域、互易对称化）。
- **若想先建立研究地图**：先做阶段 0，把两条基线曲线画出来，明确"learning 到底要超越什么"。

可进一步展开的内容（按需）：阶段 1 的网络结构与损失公式细化、阶段 0 基线实验脚本骨架、主动采样（阶段 3）的策略设计。
